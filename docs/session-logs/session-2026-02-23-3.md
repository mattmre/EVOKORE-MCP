# Session Log: 2026-02-23 (Part 3) - Voice Sidecar & Cross-CLI Sync

## Objectives Achieved

1. **Voice Sidecar** (`src/VoiceSidecar.ts`): Built a standalone WebSocket server on `ws://localhost:8888` that receives text from Claude Code hooks, streams it to ElevenLabs WebSocket TTS API, accumulates base64 audio chunks, writes to a temp MP3 file, and plays via platform audio player (PowerShell on Windows, afplay on macOS, mpv/aplay on Linux). Supports persona selection, hot-reload of `voices.json`, and optional ffmpeg `atempo` post-processing for speeds >2x.

2. **Persona Configuration** (`voices.json`): Created persona-to-voice mapping for 6 agent roles (orchestrator, researcher, architect, implementer, tester, reviewer) using well-known ElevenLabs default voices (Rachel, Domi, Bella, Antoni, Elli, Josh). Each persona overrides only fields that differ from the default config via spread merge. Default model: `eleven_turbo_v2_5` for fastest generation.

3. **Voice Hook** (`scripts/voice-hook.js`): Minimal (~20 line) Claude Code stop hook that reads stdin JSON, extracts `response.text`, forwards to sidecar via WebSocket, and silently fails if sidecar is not running. Follows same pattern as `scripts/status.js`.

4. **Cross-CLI Config Sync** (`scripts/sync-configs.js`): Utility that auto-detects installed AI CLIs (Claude Code, Claude Desktop, Cursor, Gemini CLI), reads their config files, and merges only the `evokore-mcp` server entry. Never overwrites other servers. Supports `--dry-run` and targeting specific CLIs. Tested: detects Claude Code, Claude Desktop, and Gemini CLI on this Windows machine.

5. **Package Updates**: Added `ws` runtime dep, `@types/ws` dev dep, and 4 npm scripts (`voice`, `voice:dev`, `sync`, `sync:dry`).

6. **Documentation**: Extended `docs/USAGE.md` Section 4 with sidecar setup, persona config, 3-layer speed tuning guide, and cross-CLI sync usage. Updated `CLAUDE.md` with sidecar and sync learnings.

## Key Decisions

- VoiceSidecar is standalone (never imported by `index.ts`) — runs as a separate process via `npm run voice`.
- Audio playback uses temp files + platform players rather than native audio bindings (no additional native deps).
- `voices.json` hot-reloads on each WebSocket connection (no restart needed to swap voices).
- Cross-CLI sync merges only the `evokore-mcp` entry — safe to run repeatedly without data loss.
- Gemini CLI config is manual-only (no writable config file format), so sync prints the `gemini mcp add` command.

## Technical Notes

- ElevenLabs WebSocket streaming API URL: `wss://api.elevenlabs.io/v1/text-to-speech/{voiceId}/stream-input?model_id=...&output_format=...`
- Initial message sends `xi_api_key` + `voice_settings`; subsequent messages send text chunks; empty text signals end of stream.
- `atempo` filter chains for speeds >2x: each pass maxes at 2.0, so 3x = `atempo=2.0,atempo=1.5`.
- Windows `Start-Process -Wait` blocks until the media player closes, ensuring sequential playback.
