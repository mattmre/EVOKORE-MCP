# Session Log: 2026-02-23 (Part 4) - Claude Code Hooks System

## Objectives Achieved

1. **Damage Control** (`scripts/damage-control.js`, PreToolUse hook): Security auditor that loads rules from `damage-control-rules.yaml` and enforces 4 layers of protection:
   - Dangerous command regex matching (15 patterns: `rm -rf`, `git push --force`, `DROP TABLE`, fork bombs, `npm publish`, etc.)
   - Zero-access path blocking (`.env`, `.ssh/`, credentials, secrets)
   - Read-only path enforcement (`node_modules/`, `dist/`, `.git/`, lockfiles) for Edit/Write tools
   - No-delete path protection for critical config files (`package.json`, `CLAUDE.md`, `tsconfig.json`, etc.)
   - Uses `ask` mode (exit 0 + JSON) for recoverable operations, hard block (exit 2) for destructive ones. Fail-open if rules file missing.

2. **Purpose Gate** (`scripts/purpose-gate.js`, UserPromptSubmit hook): 3-state session intent tracker:
   - First prompt: injects `additionalContext` asking user to declare session goal
   - Second prompt: records purpose from user's message
   - Subsequent prompts: injects purpose reminder to keep sessions focused
   - State persisted in `~/.evokore/sessions/{session_id}.json`

3. **Session Replay** (`scripts/session-replay.js`, PostToolUse hook + `scripts/session-replay-view.js` CLI viewer):
   - Logs every tool use as JSONL to `~/.evokore/sessions/{session_id}-replay.jsonl`
   - Per-tool summarization (command text for Bash, file_path for Edit/Write/Read, patterns for Grep/Glob)
   - CLI viewer renders ANSI-colored timeline table, supports `--latest` and `--json` flags
   - Added `npm run replay` script shortcut

4. **TillDone** (`scripts/tilldone.js`, Stop hook + CLI):
   - Hook mode: blocks session stop (exit 2) if incomplete tasks remain
   - CLI mode: `--add`, `--done`, `--toggle`, `--list`, `--clear` with `--session` flag
   - Task state in `~/.evokore/sessions/{session_id}-tasks.json`

5. **Hook Wiring** (`.claude/settings.json`): All 4 hooks registered with appropriate matchers and timeouts.

6. **Rules Definition** (`damage-control-rules.yaml`): 15 dangerous command patterns, 11 zero-access paths, 7 read-only paths, 7 no-delete paths.

7. **Documentation**: Updated `CLAUDE.md` with full hooks section including tilldone CLI usage. Updated `package.json` with replay script.

## Verification

- `damage-control.js`: Blocks `rm -rf /` (exit 2), allows `git status` (exit 0), prompts for `git push --force` (ask mode with JSON output)
- `purpose-gate.js`: All 3 state transitions verified (ask -> record -> remind)
- `session-replay.js`: Appends valid JSONL entries with timestamps and summaries
- `tilldone.js`: CLI add/done/toggle/list/clear all work. Hook blocks on incomplete tasks (exit 2), allows when all done (exit 0)
- All scripts follow fail-safe conventions (exit 0 on unexpected errors)

## Design Decisions

- **Inspired by [disler/pi-vs-claude-code](https://github.com/disler/pi-vs-claude-code)**: Ported 4 key features (security auditing, intent tracking, task discipline, session logging) from Pi Agent's extension system to Claude Code's native hook architecture.
- **Fail-open everywhere**: Every hook script wraps all logic in try/catch with `process.exit(0)` fallback. Missing rules files, malformed YAML, or any unexpected error allows the operation to proceed.
- **CommonJS + no new dependencies**: All scripts use `#!/usr/bin/env node` CommonJS (matching `voice-hook.js` and `status.js`). Only dependency is `yaml` package which was already installed.
- **Unified state directory**: All session state lives in `~/.evokore/sessions/`, logs in `~/.evokore/logs/`, matching existing `~/.evokore/cache/` pattern from `status.js`.
- **Session ID sanitization**: All scripts use `id.replace(/[^a-zA-Z0-9_-]/g, '_')` to prevent path traversal.
- **Windows path normalization**: `damage-control.js` normalizes backslashes to forward slashes for cross-platform rule matching.
